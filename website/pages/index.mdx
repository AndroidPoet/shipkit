**Shipkit** is a small, single-binary Go CLI that turns scattered mobile release
work into one command surface. You run `shipkit` to install the provider CLIs,
check readiness, generate CI, and ship — while the specialist tools keep owning
their own APIs.

```bash
shipkit guide      # answer a few questions, get the shortest setup path
shipkit install    # install gpc, rc, and asc under the hood
shipkit doctor     # check that your tooling is ready
shipkit agent --json   # deterministic context for AI agents and scripts
```

## Why Shipkit

- **One workflow, many providers.** Android releases, iOS releases, and
  subscriptions normally live in three different CLIs. Shipkit makes them feel
  like one product.
- **It does not replace the specialists.** Shipkit owns the workflow; the
  provider CLIs (`gpc`, `rc`, `asc`) own their APIs. Drop down to them anytime.
- **AI-agent-friendly by design.** `shipkit agent --json` emits deterministic,
  parseable context — no AI API calls, no API key, no telemetry, no hidden
  network calls.
- **Small enough to audit.** A focused Go codebase with clear boundaries and no
  duplicate provider clients.
- **Repeatable releases.** `shipkit ci github` generates a reviewable GitHub
  Actions workflow so every app ships the same way.
